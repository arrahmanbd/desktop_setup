Setup Git User Information

Configure your Git user name and email globally:

git config --global user.name "arrahmanbd"
git config --global user.email "arrahman@mail.com"

Enable Credential Storage

To save your Git credentials for future use:

git config --global credential.helper store

Note: When cloning a repository or performing other Git operations, use a token instead of a password. The credential will be saved automatically.

Basic Git Commands

Clone a Repository

Clone a repository to your local machine:

git clone http://example.com/repo.git

Pull Latest Changes

Fetch and merge changes from the remote repository:

git pull

Initialize a New Local Repository

Start a new Git repository in the current directory:

git init

Stage Changes

Add changes to the staging area:

git add .

Or add a specific file:

git add fileName.txt

Check Status

View the current status of your repository:

git status

View commit history in a compact format:

git log --oneline

Commit Changes

Commit your staged changes with a message:

git commit -m "Your commit message"

Push Changes

Push your local commits to the remote repository:

git push origin master

Increase Post Buffer Size for Large Pushes

If you encounter errors with large pushes, increase the buffer size:

git config http.postBuffer 524288000

Unset Post Buffer Size (If Needed)

To revert the buffer size change:

git config --global --unset http.postBuffer

Save Credentials

Your credentials are stored in:

~/.git-credentials

Handling Large Files

Install Git LFS (Large File Storage)

To manage large files, install Git LFS:

For Arch Linux:

sudo pacman -S git-lfs

Track Large Files

Track specific file types with Git LFS:

git lfs track "*.fileextension"

Example for PDF files:

git lfs track "*.pdf"

Add the file(s) to the staging area:

git add fileName.pdf

Commit and Push Large Files

Commit and push your changes as usual:

git commit -m "Your commit message"
git push origin master

Working with Branches

Create a New Branch

Create a branch from a specific commit hash:

git branch branch_name <commit-hash>

Or create a branch using a symbolic reference:

git branch branch_name HEAD~3

Create and Switch to a New Branch

Create a new branch and switch to it in one command:

git checkout -b branch_name <commit-hash or HEAD~3>
